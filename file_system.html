File system and classes auto-loading

The file system is a simplified version of Kohana's Cascading File System.

At the root of the installation directory, along with the bootstrap.php file, you'll find two directories : user and system.

+--glue/
     +--system/
     |    +--classes/
     |         +--db/
     |	       +--orm/
     +--user/
     |    +--classes/
     |         +--db/
     |	       +--orm/
     +--bootstrap.php

The system directory contains the files of the library. The user directory is where you may put your own files. Each time the library needs a file located at some/path/file.php (for example to auto-load a class - see below) it will first look for some/path/file.php in the user directory. If the file cannot be found there, it will then look for some/path/file.php in the system directory. This means that you can override any system file by creating a file of the same name at the same location in the user directory.

The library comes with its own class auto-loader. It will attempt to load any classes in the \Glue\DB and \Glue\ORM namespaces. A class named \Glue\DB\My_Class is expected to be located in the path db/classes/my/class.php. Classes are first looked for into the user directory, and only after that in the system directory, allowing you to override any system class with your own copy, as explained above.

The convention regarding inheritance is to always call A_B a class B that inherits from a class A. Thus the class Fragment_Aliased_Table inherits from the class Fragment_Aliased that itself inherits from the class Fragment. This doesn't have any particular importance but it helps making the code more readable.
