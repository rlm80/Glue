Items and builders

A recurring pattern throughout the library is the use of items and builders.

Most parts of an SQL query are lists of elements. A boolean expression is a list of boolean operands, a join expression is a list of join operands, and so on. Each of these lists is represented as a pair of classes :

  • an item class : represents the elements that make up the list and provides getter/setters to access their data,
  • a builder class : represents the list and provides a fluent interface to build it.

Builder classes are located in the classes/system/db/fragment/builder/ folder. They all extend the \Glue\DB\Fragment_Builder class and thus share some common methods :

  • ->pop() : removes last item,
  • ->reset() : removes all items,
  • ->is_empty() : whether the list is empty,
  • ->first() : returns first item,
  • ->last() : returns last item,
  • ->children() : returns the items array by reference.
