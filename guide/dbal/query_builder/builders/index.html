Items and builders

An SQL query is made up of lists of elements : boolean expressions, join expressions, order by, group by, select lists, insert lists, update lists and values.

Each of these lists is represented as a pair of fragment classes : an item class and a builder class. Even though items and builders are not very useful by themselves, their interfaces show up through the queries API, so it is necessary to understand them.

Item classes

Item fragments represent the elements that make up the lists and provide getter/setters to access their data. Item classes are located in the classes/system/db/fragment/item/ folder.

Builder classes

Builder classes represent the lists and provide chainable methods to build them. Builder classes are located in the classes/system/db/fragment/builder/ folder. All builders extend the \Glue\DB\Fragment_Builder class and thus share some common methods :

  • ->pop() : removes last item,
  • ->reset() : removes all items,
  • ->is_empty() : whether the list is empty,
  • ->first() : returns first item,
  • ->last() : returns last item,
  • ->children() : returns the items array by reference.
