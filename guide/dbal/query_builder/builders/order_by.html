Order by

Factory : DB::orderby()

Represents a comma separated list of parentheses enclosed SQL expressions, followed by an ordering specification : ("mytable"."mycolumn1" + "mytable"."mycolumn2" * 3) ASC, ("mytable"."mycolumn3") DESC, ...

Item class

Item : Fragment_Item_Orderby

The item is a fragment-order pair. The order can be either DB::ASC or DB::DESC. It has 4 methods :

  • ->ordered() : fragment getter/setter,
  • ->order() : order getter/setter,
  • ->asc() : same as ->order(DB::ASC),
  • ->desc() : same as ->order(DB::DESC).

Builder class

Builder : Fragment_Builder_Orderby

The builder has only one method (returns $this) : ->orderby(). It adds a list of SQL expressions to the order by. It can take any number of parameters, each of them can be either :

  • a string of pseudo SQL (DB::ASC is assumed),
  • a fragment (DB::ASC is assumed),
  • an array (string of pseudo SQL, order)
  • an array (fragment, order)

Examples

{% highlight php %} orderby("`column2`", array("column3", db::DESC)); // Output SQL : echo db::cn()->compile( $f ); // Prints ("column1" + "column2" * 5) ASC, ("column2") ASC, (column3) DESC // Usage with table fragment : $t = db::table('mytable', 'myalias'); $f = db::orderby("$t->column1 + $t->column2 * 5")->orderby($t->column2, array($t->column3, db::DESC)); // Output SQL : echo db::cn()->compile( $f ); // ("myalias"."column1" + "myalias"."column2" * 5) ASC, ("myalias"."column2") ASC, ("myalias"."column3") DESC ?> {% endhighlight %}
